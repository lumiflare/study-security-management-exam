# ネットワークセキュリティ

## DMZ（非武装地帯）
- 外部と内部の間の緩衝領域
- DMZに置く: Web/メール/DNSサーバ（外部公開するもの）
- 内部LANに置く: 業務サーバ、ファイルサーバ、社員PC

## ファイアウォール
| 種類 | 検査対象 |
|------|---------|
| パケットフィルタリング | IPアドレス、ポート番号 |
| ステートフルインスペクション | 上記＋通信状態（セッション） |
| アプリケーションゲートウェイ | 通信内容まで検査（プロキシ動作） |
- FWは通信の中身を見ない → SQLインジェクション等はWAFが必要

## IDS/IPS
- IDS: 侵入「検知」→ 通知のみ
- IPS: 侵入「防止」→ 検知＋遮断
- シグネチャ型: 既知パターン照合（誤検知少、未知に弱い）
- アノマリ型: 異常検知（未知に強い、誤検知多い）

## WAF
- Webアプリ専用のFW、HTTPリクエストの中身を検査
- SQLインジェクション、XSS、OSコマンドインジェクション等を防御

## VPN
- IPsec-VPN: 拠点間接続、ネットワーク層、専用ソフト必要
- SSL-VPN: リモートアクセス、ブラウザで利用可能

## TLS/SSL
- ハイブリッド暗号方式を利用
- サーバ証明書でサーバの正当性を確認
- HTTPS = HTTP + TLS（ポート443）

## 無線LANセキュリティ
- WEP: 脆弱（RC4）→ 使用禁止
- WPA: 非推奨（TKIP）
- WPA2: 標準（AES/CCMP）
- WPA3: 最新（AES/SAE）

## プロキシ
- フォワードプロキシ: 内部→外部（社内からのWeb閲覧中継）
- リバースプロキシ: 外部→内部（外部アクセスを内部サーバに転送）

## 防御機器の守備範囲
- FW: ネットワーク層（IPアドレス/ポート）
- IDS/IPS: ネットワーク〜トランスポート層
- WAF: アプリケーション層（HTTP内容）
