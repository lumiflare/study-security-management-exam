# 暗号・認証技術

## 暗号方式の比較
| | 共通鍵暗号 | 公開鍵暗号 |
|---|----------|----------|
| 鍵 | 同じ鍵で暗号化・復号 | 公開鍵と秘密鍵のペア |
| 速度 | 高速 | 低速 |
| 鍵の数 | n(n-1)/2 | 2n |
| 代表 | AES | RSA |
| 用途 | 大量データの暗号化 | 鍵交換、デジタル署名 |

## ハイブリッド暗号
- 共通鍵をランダム生成→公開鍵暗号で共通鍵を送付→以降は共通鍵で通信
- HTTPS等で使用

## ハッシュ関数
- 一方向性（逆算不可）、衝突耐性、固定長出力
- SHA-256が現在の標準、MD5/SHA-1は非推奨
- 用途: パスワード保存、改ざん検知、デジタル署名
- ハッシュは暗号化ではない（復号できない）

## デジタル署名
- 送信者の秘密鍵で署名、送信者の公開鍵で検証（暗号化通信と逆方向）
- 保証するもの: 完全性、否認防止
- 保証しないもの: 機密性

## PKI（公開鍵基盤）
- CA（認証局）: デジタル証明書を発行、公開鍵の正当性を保証
- RA（登録局）: 申請受付・本人確認
- CRL: 失効した証明書の一覧
- OCSP: 証明書の有効性をリアルタイム確認

## 認証の3要素
| 要素 | 例 |
|------|-----|
| 知識 | パスワード、PIN、秘密の質問 |
| 所持 | ICカード、スマホ、トークン |
| 生体 | 指紋、虹彩、静脈、顔 |

## 多要素認証
- 異なる要素を2つ以上組み合わせる
- パスワード+秘密の質問 = 両方「知識」なので多要素ではない

## バイオメトリクス評価指標
- FRR（本人拒否率）: 本人を拒否してしまう率
- FAR（他人受入率）: 他人を受け入れてしまう率
- トレードオフ関係: 厳しくするとFRR↑FAR↓

## 試験注意点
- 暗号化通信: 受信者の公開鍵で暗号化
- デジタル署名: 送信者の秘密鍵で署名（方向が逆）
- 二段階認証 ≠ 二要素認証（同じ要素の二段階は多要素ではない）
